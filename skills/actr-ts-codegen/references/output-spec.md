# Output Specification

## File Naming
- Package name `echo` -> `echo.pb.ts`, `echo.client.ts`.
- Package name with dots `foo.bar` -> `foo-bar.pb.ts`, `foo-bar.client.ts`.
- JS output mirrors TS output with `.js` extension.

## Protobuf Codec Files
- Header comment:
  - `// DO NOT EDIT.`
  - `// Generated from <proto paths>`
- Exported interfaces for each message type.
- Exported codec objects named `<PackagePascal>_<MessageName>`.

## Client Helper Files
- Header comment:
  - `// DO NOT EDIT.`
  - `// Generated by actr-ts-codegen.`
- Export route key constants in upper snake case:
  - `<METHOD>_ROUTE_KEY` or `<SERVICE>_<METHOD>_ROUTE_KEY` when method names collide.
- Export `encode<MsgName>` and `decode<MsgName>` helpers.
  - Single-field requests produce `encode<MsgName>(value)` convenience functions.

## Local Actor
- Header comment:
  - `// DO NOT EDIT.`
  - `// Generated by actr-ts-codegen.`
- `ROUTES` table includes `routeKey` and `targetType` (`{ manufacturer, name }`).
- `dispatchLocalActor` forwards to `ctx.callRaw` with `RPC_PAYLOAD_TYPE` set to `0`.
